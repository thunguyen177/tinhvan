The packages problem

- If possible, import all the package that contain scripts right from the start. I remember importing unity in-app purchase package, and then I also import some package that creates sharing to social media. More than 100 errors appeared right after that. Sometimes, package conflict can be solved by deleting the newly imported package that causes the conflict and restart Unity. Yet, that's not apply when I also did some thing else. So, I created another empty project and import files from the old one, instead of debugging over 100 errors.
- A maybe faster way to import assets into Unity is by dragging files into Unity. The reason is you cannot directly import a whole folder into Unity by right mouse -> import new assets.
