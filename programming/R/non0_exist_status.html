Fixed: installation of a package had non-zero exit status

This error is pretty common in R. Here, I illustrate how to fix it with an example of installing 'lcmix' package:
On https://rdrr.io/rforge/lcmix/, they give the instruction how to install package lcmix as 
> install.packages("lcmix", repos="http://R-Forge.R-project.org")
    ....
Warning in install.packages :
  installation of package ‘lcmix’ had non-zero exit status
You may try again with 
       > install.packages("lcmix", repos="http://R-Forge.R-project.org", dependencies = T)
But this works only if the dependencies is also in the same repository with the package that you want to install. So, if you look more carefully, you'll see more details: 
       ERROR: dependency ‘R.methodsS3’ is not available for package ‘matrixStats’
        * removing ‘/home/thu/R/x86_64-pc-linux-gnu-library/3.2/matrixStats’
        Warning in install.packages :
          installation of package ‘matrixStats’ had non-zero exit status
        ERROR: dependencies ‘R.methodsS3’, ‘matrixStats’, ‘nnls’ are not available for package ‘lcmix’
        * removing ‘/home/thu/R/x86_64-pc-linux-gnu-library/3.2/lcmix’
        Warning in install.packages :
          installation of package ‘lcmix’ had non-zero exit status
To be more specific, here the packages matrixStats, nnls, R.methodsS3 is available on CRAN, not R-Forge. So, all you need to do is install these packages manually 
    install.packages("matrixStats")
    install.packages("nnls")
    install.packages("R.methodsS3")
Then, install lcmix from R-Forge again with: 
 install.packages("lcmix", repos="http://R-Forge.R-project.org")
or 
 install.packages("lcmix", repos="http://R-Forge.R-project.org", dependencies = T)
Result: 
    The downloaded source packages are in
     ‘/tmp/Rtmpa9DRqE/downloaded_packages’
